## 关键业务逻辑

### 认证流程

- 教师登录使用 `sha256()` 加密密码
- Token 通过 `Token::set()` 存储，有效期30天
- `Teacher::initTeacher()` 自定义了Token初始化逻辑（覆盖基类的Auth）
- 所有 stu 控制器设置 `$noNeedLogin = '*'` 和 `$noNeedRight = '*'`（免登录免鉴权）

### 积分系统

- 积分记录存储在 `fa_stu_score` 表，包含 `student_id, class_id, week, day_of_week, score, reason`
- 学生表 `fa_stu_student` 维护 `total_points`（累计积分）和 `current_points`（当前可用积分）
- 批量撤销积分：伪删除（status=0）+ 反向调整学生积分
- 小组积分通过 `fa_stu_group` 表的 `weekly_score_json` 字段存储

### 宠物游戏

- 宠物有 `energy`（体力）和 `mood`（心情）两个核心属性
- 时间衰减机制：每小时体力-5，心情-5（`GamePet::applyTimeDecay()`）
- 成长阶段：幼年期→成长期→成熟期→完全体
- 喂养满体力3次触发进化
- 背包系统：每个小组独立的食物和玩具库存

### 奖品系统

- 4种类型：1=学生奖励, 2=学生周奖, 3=小组奖励, 4=小组周奖
- 兑换时扣减学生 `current_points`
- 小组发放按排名分配数量：第1名5份，第2名4份，第3名3份，其余1份
- 兑换记录状态：1=已发放, 2=待处理

**功能**：

- 支持批量操作（传入数组或单个对象）
- 等待所有异步调用结束
- 统计成功/失败数量

  - 全成功：`x个奖品已发放`
  - 部分成功：`x个成功，y个失败`

**代码**：

```ts
const updateExchangeStatus = async (row: any[] | object) => {
    const items = Array.isArray(row) ? row : [row]
    // 统计成功/失败
    let successCount = 0, failCount = 0
    for (const item of items) {
        try {
            const res = await updateStatus({ id: item.id, status: 1 })
            if (res.code === 1) successCount++
            else failCount++
        } catch { failCount++ }
    }
    // 智能提示
    if (successCount > 0 && failCount === 0) {
        ElMessage.success(`${successCount}个奖品已发放`)
    } else if (successCount > 0 && failCount > 0) {
        ElMessage.warning(`${successCount}个成功，${failCount}个失败`)
    } else {
        ElMessage.error('发放失败，请重试')
    }
    // 刷新列表
    if (successCount > 0) await openExchangeRecordDialog()
}
```
